# 安心办初赛评分对照表

## 1. 评分维度与项目体现方式

### 1.1 创新性 30%

我们要体现的不是"技术堆砌"，而是两个更核心的创新点：

- 需求创新：不是替老人办业务，而是帮助老人本人看懂医保、银行、政务 App 的复杂说明。
- 解决思路创新：把"说明文本理解"拆成通俗解释、步骤卡、安全提醒三段结构。
- 技术创新：用 AI 生成结构化结果，再结合后端关键词兜底，把"可理解"和"可控风险"结合起来。

### 1.2 实用性 30%

我们要体现：

- 场景洞察真实：老人确实会被说明长文、验证码、转账、收费、链接等内容困扰。
- 解决效果明确：用户能从"看不懂说明"变成"先理解，再按步骤看，再看到风险提醒"。
- 落地可行：前端静态页 + 轻量后端服务，路径简单、适合演示和快速落地。

### 1.3 完成度 20%

我们要体现：

- 核心链路完整：选择场景 → 粘贴说明 → AI 生成通俗解释 / 步骤卡 / 安全提醒 → 前端展示。
- 用户体验清楚：普通场景先看解释，高风险场景先看提醒，步骤卡一屏一步。
- 价值传递明确：评委能一眼看出，这个产品是在"降低老人数字服务理解门槛"。

### 1.4 美观度 / 设计体验 20%

我们要体现：

- 视觉设计清楚：大字、高对比、少按钮、移动端优先。
- 交互友好：每页只做一件事，主要信息优先展示。
- 老人可读：不是普通后台页面，而是面向老人本人使用的阅读型界面。

## 5. 安全结论卡升级（差异化强化）

### 5.1 升级目标

把安心办从"AI 解释说明"强化成"老人安全操作陪跑"：老人进入结果页第一眼，先得到一个明确的"能不能继续操作"结论，再看解释和步骤。

### 5.2 实现内容

- 结果页顶部新增安全结论卡，根据 AI 返回的 `riskLevel` 分三档：
  - low：绿色"可以继续办 / 按下面步骤来就行"
  - medium：琥珀色"先看清楚，再继续 / 有些地方要注意"
  - high：红色"先别点，可能有风险 / 先找家人或官方渠道确认"
- 图标使用内联 SVG（勾、感叹、叉），不使用纯文本符号。
- 后端 `buildPrompt` 新增三档 `riskLevel` 判定标准说明，让 AI 输出更稳定。
- 后端新增注意事项窄兜底：当 AI 返回 low 但文本出现证件原件、窗口、现场核对等办事注意词时，稳定升为 medium。
- 不改接口结构，不加新功能，不引入新依赖。

### 5.3 验证结果

- 医保普通说明：绿色"可以继续办" ✅
- 银行高风险说明：红色"先别点，可能有风险"，风险提醒条仍正常 ✅
- 政务 medium 样例：琥珀色"先看清楚，再继续" ✅（注意事项兜底稳定生效）
- 通用入口：按 `riskLevel` 正常显示 ✅
- 移动端 390×844：结论卡不挤压 ✅
- 控制台无 JS 错误 ✅

### 5.4 评分价值

- **创新性 30%**：从"解释说明"升级为"安全操作陪跑"，先给结论再给细节，产品定位差异化更明确，不再是通用 AI 解释工具。
- **实用性 30%**：老人第一眼就能知道"能不能继续办"，在高风险场景下显著降低误操作风险，符合"保护老人安全"的核心价值。
- **完成度 20%**：三档风险等级可视化覆盖全部场景（医保/银行/政务/通用），移动端布局完整，high 档结论卡、风险提醒条和确认按钮同时正常。
- **美观度 20%**：绿/琥珀/红三色可视化 + 内联 SVG 图标，删除重复的底部安全提醒卡，页面更短、更清楚。
